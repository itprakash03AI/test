Welcome to QueryStudio

Your Enterprise Data Platform — Demo Talking Points — April 24, 2026

QueryStudio is an enterprise self-service data platform. End users connect to any data source — on-prem S3 (Dell ECS, MinIO), AWS S3, Snowflake, Oracle, Databricks, Trino, or local files — and query data visually without writing SQL. Partition-aware execution on Iceberg tables turns 45-minute scans into 5-second queries. Results run asynchronously and persist for 7 days.

Built-in scheduling, saved queries, published data views, feed generation (S3/Oracle/SFTP), dashboards, business reports, data catalog, role-based access, and full audit trail. Runs on OpenShift (on-prem) and AWS EKS (cloud). No vendor lock-in.

DEMO 1

Create Source Connections

5–7 min
Slides: "Connect to Any Data Source" + "Setting Up a Connection"

1. On-Prem S3 (Dell ECS / MinIO)

  • Click "New Connection" → select "S3 (On-Premises)"
  • Enter: endpoint URL, bucket name, access key, secret key
  • Toggle "Path-style access" ON (required for on-prem S3)
  • Click "Test Connection" → green checkmark = success
  • Click "Save" → connection appears in sidebar

"We support on-prem S3 natively. No special configuration, no gateway. Direct connection to your Dell ECS or MinIO storage."

2. Cloud S3 (AWS)

  • Click "New Connection" → select "AWS S3"
  • Enter: bucket name, region (dropdown), access key ID, secret key
  • OR: select "IAM Role" auth mode (no keys needed in AWS environment)
  • Click "Test Connection" → validates bucket access

"Same interface for cloud and on-prem. End users don't need to know where the data lives."

3. Local Parquet Files

  • Click "New Connection" → select "Local Folder"
  • Browse or enter path to parquet directory
  • OR: Drag-and-drop parquet files into "Upload" connection
  • Files appear immediately in the sidebar file browser

"For quick analysis, end users can upload parquet/CSV/Excel files directly. Self-service."

Transition: "Now that we have connections, let's query some data..."

Partitions & Query Execution

7–10 min
Slides: "Partition Pruning" + "Same Data, Same Question" + "Visual Query Builder"

1. Open the Query Builder

  • Click on an S3 connection in the sidebar
  • Browse the folder structure → find an iceberg table
  • Click the table name → columns and partitions auto-detected

"QueryStudio automatically detects iceberg table format and reads partition metadata. No manual configuration."

2. Explain Partitions (use slide diagram)

  • "Iceberg tables split data into partitions — typically by date."
  • "A table with 2 years of daily data has ~730 partitions."
  • "Without partition pruning, querying this table reads ALL partitions — potentially terabytes of data."
  • "With partition pruning, we only read the partitions you need."
  • SHOW the partition selector panel in the UI:
  • Date-aware quick filters: "Last 7 days", "Last 30 days", "This Month"
  • Manual selection: check/uncheck specific partition values
  • "Select All" / "None" / "Invert" toggles

"This is the difference between a 45-minute query and a 5-second query. Same data, same question — just smarter execution."

3. Build a Query

  • Select columns from the column picker (drag or checkbox)
  • Add a filter: e.g., amount > 1000
  • Select partitions: e.g., "Last 30 days"
  • Show the SQL preview panel: "This is the SQL that will execute"

"End users build queries visually. No SQL knowledge required. But the SQL is always visible for those who want it."

4. Execute the Query

  • Click "Run Query"
  • Show the progress indicator (real-time via WebSocket)
  • Results appear in the data grid
  • Show row count, execution time, data scanned

"Query completed in X seconds, scanning only Y partitions out of Z total. That's partition intelligence at work."

Transition: "Now let's save this query for future use..."

Save Query

2–3 min
Slide: "Save & Reuse Queries"

1. Save the Query

  • Click "Save Query" button
  • Enter a meaningful name: e.g., "GL Transactions — Last 30 Days"
  • Add description: "Monthly GL transaction summary for finance team"
  • Click "Save"

"Queries are saved with their full configuration — tables, columns, filters, partitions. Not just the SQL text."

2. Show Saved Queries

  • Open the "Saved Queries" panel in the sidebar
  • Show the list of previously saved queries
  • Click a saved query → entire query builder state restores

"End users don't start from scratch each time. Build once, reuse forever. Saved queries can also be scheduled for automated execution."

3. Mention Additional Capabilities (don't demo, just mention)

  • "Saved queries can be published as 'Published Views' — read-only data endpoints that other teams can consume."
  • "They can also be used as feed sources for automated data delivery."
Transition: "But what about large queries that take time?"
DEMO 4

See Results Later

3–5 min
Slide: "Submit & Walk Away"

1. Submit a Large Query

  • Select a larger dataset (more partitions, more rows)
  • Click "Run Query"
  • Show: progress bar with percentage and row count

"For large queries, execution runs in the background. You don't need to keep the browser open."

2. Close the Browser / Navigate Away

  • "Let me close this tab and come back later..."
  • Re-open QueryStudio

"The query continues running on the server. Results are persisted for 7 days."

3. Show Execution History

  • Navigate to "Execution History" page (or Downloads)
  • Show the list of past executions with status:
  • Green = Completed (ready to download)
  • Yellow = Running (still processing)
  • Red = Failed (with error message)
  • Click on a completed execution → see results in data grid

"Full visibility into what ran, when, how long, how many rows. Complete audit trail."

4. Download Results

  • Show "Download" button → CSV export
  • Show "Export" dropdown → Excel, JSON options

"Results can be downloaded in multiple formats. Share with anyone."

Tips for the Demo

  1. Pre-load data: Make sure your demo S3 bucket has a good iceberg table with date partitions. Ideally GL or trading data the audience recognizes.
  2. Pre-create one connection: Have at least one working connection ready. Create the second one live during demo.
  3. Keep queries fast: Use small date ranges (7 days) so results come back in under 10 seconds.
  4. Have a fallback: If live demo breaks, have screenshots ready. "Let me show you what this normally looks like..."
  5. Focus on value, not features: Don't say "we added partition pruning." Say "queries that took 45 minutes now take 5 seconds."
  6. Know your audience: If finance people — use GL/accounting examples. If trading — use trade/position examples. Use their language.
